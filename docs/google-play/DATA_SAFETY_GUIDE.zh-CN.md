# Data Safety 填写建议（qbremote）

本说明基于当前代码实现：
- 仅申请 `INTERNET` 权限
- 无 Firebase / 无广告 SDK / 无崩溃统计 SDK
- 用户名与密码仅本地保存（密码使用 `EncryptedSharedPreferences`）
- 网络请求只发往用户自行配置的 qBittorrent WebUI 地址

## 建议填写口径（推荐）

### 1) Data collected（是否收集数据）

建议选择：`No`（不收集）

说明口径：
- 开发者不接收、存储或共享用户数据
- 应用仅作为客户端与用户自建/自管的 qBittorrent 服务直接通信

### 2) Data shared（是否共享数据）

建议选择：`No`

### 3) Security practices（安全实践）

- 数据删除：`Yes`（用户可在应用内删除服务器配置）
- 传输加密：谨慎填写
  - 应用支持 HTTPS，也允许 HTTP（局域网场景）
  - 不建议声明“所有数据始终加密传输”

## 若审核要求更保守申报（备选口径）

若你遇到审核质疑，可改为声明：
- 个人信息 -> 用户名（用于账号登录功能）
- 敏感信息 -> 密码（用于账号登录功能）
- 用途：`App functionality`
- 不用于广告、不用于分析、不用于出售
- 不共享给开发者控制的第三方

## 填写前自查

- [ ] 确认未引入统计/广告/崩溃采集 SDK
- [ ] 确认隐私政策内容与 Data Safety 描述一致
- [ ] 确认审核说明里写清楚“连接目标是用户自定义服务器”
- [ ] 若启用 HTTP，避免在表单中做“全链路加密”绝对承诺
